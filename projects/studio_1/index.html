Studio 1: Hangman

Rules of the Game:

  • A random word is selected as the word to discover
  • The player is shown a blank space for each letter of the random word
  • The player takes turns guessing letters that the word might contain; if the word contains the letter, every instance of the letter in the word is revealed; if the word does not contain the letter, the number of guesses is decremented
  • The player starts with 10 guesses

Prompt

The goal of this studio is to make a simple game of Hangman. The player will make guesses as to what the word is one character at a time until either the full word is revealed or the player runs out of guesses and loses the game (upon which the word should also be fully revealed).

Details

For the "activity_main.xml" file, there will need to be one surrounding layout; inside this layout, there should be a TextView that contains the random word, an EditText field to make guesses, a Button to submit those guesses (and to also function as restart button), and a TextView showing the number of remaining guesses. All four elements should have an id associated with them in order to access them from the MainActivity.kt file.

For the MainActivity.kt file, you will only need the onCreate method and two global variables, one for the number of guesses and one boolean for whether or not the game is over. You may throw the array of words below into the onCreate method, set up a random number generator to select a random index in the array. Lastly, there will need to be a click event listener for the guess/restart button that will use logic based on the global boolean to designate what should happen. If the game is not over, then there should be a check for whether or not the letter submitted is contained within the word and update the UI. If the game is over, the following code should be executed:
