Project 2: To-Do List

Prompt

The goal of this project is to make an app that allows a user to keep track of things they need to do/accomplish. This data should be stored in a SQLite database on the device. I have not included an User Interface components in this lab because I want you to experiment a little in how you think about displaying the data to a user.

Requirements

  • MainActivity.kt holds a list of To-Do items
  • CategoryActivity.kt holds a list of Categories
  • ToDoActivity.kt contains a view capable of creating/editing a to-do item
    • User should be able to give the item a title
    • User should be able to give the item a category
    • User should be able to give the item a body of detail text
    • User should be able to give the item a deadline/due date
    • User should be able to assign a level of importance
  • User should easily be able to delete a to-do item
  • User should be able to "check off" or complete a to do
  • User should easily be able to distinguish between completed and incomplete to-do items
  • User should be able to sort the list of incomplete to-do items by deadline/due date, importance, or alphabetical by title
  • Completed to-do items should be sorted by date/time completed

Submission

Each of you will create a PRIVATE GitHub repository and share it with me AND Bill Siever. This way if you also have trouble along the way it can make it easier for me to view your code remotely.
